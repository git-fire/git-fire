# Show HN launch post

## Title
Show HN: git-fire - one command to checkpoint and push all your repos

## Body (<300 words)
I built `git-fire` for one recurring problem: before context switches, risky refactors, or travel, I need a fast way to checkpoint lots of local repos without manual loops.

It's a spiritual successor to `qw3rtman/git-fire` (Bash, now abandoned). I used that tool for years, then rebuilt the concept in Go to make it safer, testable, and easier to extend.

What it does now:

- streamed scan -> backup pipeline so work starts immediately
- parallel execution with bounded workers for large repo sets
- persistent repo registry (set once, reuse)
- dry-run planning and one-screen `--status`
- secret detection guardrails before push
- structured JSON logs for audit/automation
- 250+ tests on core packages

Useful for human and agent workflows: run one command to checkpoint everything a coding session touched across repos.

Built with Go + Cobra, Viper, Bubble Tea, and Lipgloss.
No tagged release yet; first release is planned soon. For now, install from source with `go install github.com/git-fire/git-fire@main`.

Current state: alpha/MVP. Latest main also includes a Windows path handling fix that will be in the first tagged release.

Likely critical comment:
"Why not just use a shell loop?"

Suggested honest response:
A shell loop is fine for simple cases. This is for repeatability and guardrails at larger scale: dry-run planning, conflict-safe backup branches, secret warnings, bounded concurrency, and structured logs. The goal is fewer failure modes when many repos are involved.
