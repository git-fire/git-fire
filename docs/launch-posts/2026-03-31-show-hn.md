# Show HN launch post

## Title
Show HN: git-fire - open-source CLI to checkpoint and push many repos at once

## Body (<300 words)
I built `git-fire` for a simple problem: quickly checkpoint and push lots of local repos before context switches, risky changes, or machine chaos.

It's a spiritual successor to `qw3rtman/git-fire` (Bash, now abandoned). I used that version for years; this one keeps the same spirit but is rebuilt in Go for better reliability and testability.

What it does today:

- parallel execution with bounded workers (handles large repo sets, not just 2-3)
- persistent repo registry (configure once, run repeatedly)
- repeatable end-of-session checkpoint flow
- dry-run mode before making changes
- `--status` per-repo snapshot in one screen
- extensible via plugins (v0.2)
- secret detection warnings before push
- 250+ tests

It's also useful as an end-of-agent-session safety net for AI coding workflows (Cursor/Claude Code/Copilot): run one command to checkpoint everything the agent touched across repos.

Built with Go + Cobra, Viper, Bubble Tea, and Lipgloss.
Install via Homebrew, Scoop, or `go install`. MIT licensed.

Status: alpha/MVP. It works in my daily flow, but I assume rough edges exist and I'd rather hear them early.

Repo: https://github.com/git-fire/git-fire

Recommended post time (ET): Tuesday, 10:00 AM ET

Likely critical comment:
"Why not just use a shell loop?"

Suggested honest response:
A shell loop is fine for basic cases. This adds guardrails and repeatability: conflict-safe backup branches, dry-run planning, secret warnings, structured logs, and (soon) plugin hooks, all covered by 250+ tests. It's about reducing failure modes when many repos are involved.
